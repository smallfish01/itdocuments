How-to-use-Hexo

Posted on 2019-08-27 | In Others | Visitors:

Mac 系统安装Hexo简洁版

1.参考文档：
https://hexo.io/zh-cn/docs/
https://github.com/limedroid/HexoLearning

安装必须工具：

$ sudo brew install node
$ sudo npm install git
$ sudo npm install hexo-cli -g

2.创建和部署

创建新文章：
hexo new “文章名”
默认文章创建在source/_posts下，可以直接编辑文件。

部署到本地：

hexo g -d
hexo s
默认可通过浏览器http://localhost:4000进行本地预览

3.删除文章：
到目录/source/_posts下删除相应的文章
$ hexo g
$ hexo d

4.插入图片：

1).可以通过将图片统一存放在source/images文件夹下,在写文章时通过以下命令引用：
source/images/image.jpg

如果是外链地址可以直接在括号中输入完整地址即可。

2).也可以通过将图片放在每个文章自己目录中，需要安装插件：
$ npm install hexo-asset-image –save

修改配置文件_config.yml,启用：
post_asset_folder: true
如果使用外链地址可以不需要启用该项。

之后每次通过hexo new创建文章的时候都会创建与文章同名的目录，将图片放在该目录下，然后在文章中引用图片：
_posts/post_name/image.jpg

如果希望图片在文章和首页中同时显示，可以使用标签插件语法：
_posts/post_name/image.jpg

5.安装主题Next

官方文档：http://theme-next.iissnan.com/getting-started.html

1).下载主题：
$ cd 博客目录
$ git colone https://github.com/iissnan/hexo-theme-next themes/next

2).修改主题为next
$ vi _config.yml
theme: next

3).部署到服务器
$ hexo g -d
$ hexo s

6.发布文章到github.io

1).设置github
登录github，创建新的repository,例如名字为：smallfish01.github.io
创建完毕后复制链接，修改本地_config.yml,设置deploy信息：
deploy:
type: git
repository: git@github.com:smallfish01/smallfish01.github.io.git
branch: master

2).在本地电脑生成ssh key
$ cd ~
$ cd .ssh
$ ssh-keygen -t rsa -C “smallfish01”

3).在Github上设置ssh key
登录Github–Settings–SSH and GPG keys–SSH keys–New SSH Key
Title：输入自定义Title
Key:黏贴本地产生的id_rsa.pub文件内容
点击Add SSH key.

4).测试ssh key添加是否成功，在本地电脑命令行输入：
$ ssh -T git@github.com
The authenticity of host ‘github.com (13.229.188.59)’ can’t be established.
RSA key fingerprint is SHA256:nThbg6kXUpJWGl7E1IGOCspRomTxdCARLviKw6E5SY8.
Are you sure you want to continue connecting (yes/no)? yes
Warning: Permanently added ‘github.com,13.229.188.59’ (RSA) to the list of known hosts.
Enter passphrase for key ‘/Users/username/.ssh/id_rsa’:
Hi smallfish01! You’ve successfully authenticated, but GitHub does not provide shell access.

修改Github上传后文件的姓名,当你上传文章后在Github上显示的名字。
git config –global user.name “Fly.Fish”

5). 部署网站到Github
在本地输入：
hexo generate
hexo deploy

访问https://smallfish01.github.io 进行测试

6).免费图床
Telegram添加好友：@sm_ms_bot，直接将图片拖入聊天框，系统会自动产生一个图片地址，将改地址贴到文章即可。

6.其他配置

1).创建分类：
$ hexo new page categories
INFO Created: ~/knowing/source/categories/index.md
vi /knowing/source/categories/index.md,在date后添加：
type:”categories”

2).修改文章，添加分类：
例如将文章test.md 放到分类：test中，可以编辑test.md,在tags：后插入：
categories:test

$ hexo clean ##清除缓存
$ hexo g
$ hexo d

7.问题：

1).部署到github上提示：
ERROR Deployer not found: git

解决：
npm install hexo-deployer-git –save

Using-Hyper-v-Replication

Posted on 2019-08-26 | In Virtualization | Visitors:

Windows Hyper-V 部署复制功能

目的

通过Hyper-v自带的复制功能，在主服务器VM在正常运行的时候，定时自动复制数据到另一台VM，当主服务器故障时，可以很快通过切换到被复制主机恢复服务器运行。

环境

源主机地址：192.168.20.30，所在宿主机：192.168.20.9
被复制主机：192.168.20.30，所在书主机：192.168.20.8

配置步骤

1.需要配置两台Hyper-v启用复制并设置防火墙允许http/https：

2.选中要复制的虚拟机–邮件–复制，配置复制到另外一台宿主机,复制方法和时间根据实际需求调整，一般选用默认即可。

3.复制配置完成后，备机默认是关闭状态，可以在主备机上查看当前复制状态。

4.当主机故障无法连接的时候，可以启用备机进行恢复模式。选择备机–右键–复制–故障转移。启用备机提供服务。
当主机恢复后，可以关闭备机，重新启用主机提供服务。

5.使用反向复制功能。

当主机不能持续提供服务，需要切换为备机状态时，可以将当前的备机提升为主机，配置反向复制。
将主机关机，选择复制–计划的故障转移–将服务转移到备机上。

服务转移到备机后，在主机上选择反向复制：

根据提示操作即可，将当前备机提升为主机:


配置完毕后在主备机上检查确认即可。
